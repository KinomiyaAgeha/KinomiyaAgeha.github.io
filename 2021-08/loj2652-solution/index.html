LOJ2652 Zap 题解

题目要求等价于求满足 x≤a/k,y≤b/k\large x \le a/k, y \le b/kx≤a/k,y≤b/k 并且 gcd(x,y)=1\large \gcd(x,y)=1gcd(x,y)=1 的二元组 (x,y)\large (x,y)(x,y) 的数量。

设 g(a,b,k)g(a,b,k)g(a,b,k) 为满足 x≤a,y≤b\large x\le a, y\le bx≤a,y≤b 且 k∣gcd(x,y)\large k \mid \gcd(x,y)k∣gcd(x,y) 的二元组 (x,y)\large (x,y)(x,y) 的数量，不难想到

g(a,b,k)=(⌊a/k⌋)×(⌊b/k⌋)\large g(a,b,k)=(\lfloor a/k \rfloor ) \times (\lfloor b/k \rfloor) g(a,b,k)=(⌊a/k⌋)×(⌊b/k⌋)

由容斥原理，得由于 g(a,b,1)=a×b\large g(a,b,1)=a \times bg(a,b,1)=a×b，比真正的答案多了 gcd(x,y)\large \gcd(x,y)gcd(x,y) 是 2,3,5 \dots 的倍数的个数，所以要减去它们，即 g(a,b,2)\large g(a,b,2)g(a,b,2) 与 g(a,b,3)\large g(a,b,3)g(a,b,3)。

而减去后多减了 gcd(x,y)\large \gcd(x,y)gcd(x,y) 既是 222 的倍数，又是 333 的倍数的个数，即 g(a,b,6)\large g(a,b,6)g(a,b,6)，又要加上。以此类推，我们发现加减的关系正是 k\large kk 的莫比乌斯函数。

Mobios(x)\large Mobios(x)Mobios(x) 记为 μ(x)\large \mu(x)μ(x) 。

所以答案为

f(a,b)=∑i=1min(a,b)μ(i)×g(a,b,i)\large f(a,b) = \sum_{i=1}^{\min{(a,b)}} \mu(i) \times g(a,b,i) f(a,b)=​i=1​∑​min(a,b)​​μ(i)×g(a,b,i)

由于

∀i∈[x,min(a/⌊a/x⌋⌋,⌊b/⌊b/x⌋⌋)]\large \forall i \in \left[ x, \min(a/\lfloor a/x \rfloor \rfloor ,\lfloor b/ \lfloor b/x \rfloor \rfloor)\right] ∀i∈[x,min(a/⌊a/x⌋⌋,⌊b/⌊b/x⌋⌋)]

\large g(a,b,i)=(\lfloor a/i \rfloor ) \times (\lfloor b/i \rfloor) \text{ 都相同}

所以实现的时候求出 μ(x)\large \mu(x)μ(x) 的前缀和，直接处理一个区间的信息。

#include<cstdio>
#include<iostream>
using namespace std;
#define ll long long
const int N=5e4+10;
int n, a, b, k, Mobius[N];
bool v[N];
int sol() {
    int x, gx, ans=0;
    a/=k, b/=k;
    if(a>b) swap(a,b);
    for(x=1;x<=a;x=gx+1) {
        gx=min(a/(a/x),b/(b/x));
        ans+=(Mobius[gx]-Mobius[x-1])*(a/x)*(b/x);
    }
    return ans;
}
int main() {
    int i, j;
    for(i=1;i<=5e4;++i) Mobius[i]=1;
    for(i=2;i<=5e4;++i) if(!v[i]) {
        Mobius[i]=-1;
        for(j=(i<<1);j<=5e4;j+=i) {
            v[j]=1;
            if(j/i%i==0) Mobius[j]=0; else Mobius[j]*=-1;
        } 
    }
    for(i=1;i<=5e4;++i) Mobius[i]+=Mobius[i-1];
    scanf("%d",&n);
    while(n --> 0) {
        scanf("%d%d%d",&a,&b,&k);
        printf("%d\n",sol());
    }
}
