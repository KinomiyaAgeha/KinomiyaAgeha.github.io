CF1179D Fedor Runs for President 题解

发表于 2021-10-30 更新于 2021-12-19 分类于 题解 阅读次数：

树上任意两点间，有且仅有一条简单路径。

数量 \[ \large C^2_n = \frac{n (n-1)}{2} \] 对于添加边 \(\large (x,y)\)，设它们树上路径一点为 \(\large z\)。

加边后显然会多出一个简单环

设为 \(\large S_z\) 为以 \(\large z\) 为根且不含环上节点的子树大小。

则贡献为 \[ \large \frac{S_z(n-S_z)}{2} \] 总贡献 \[ \large \frac{\sum_{z \in (x,y)} S_z(n-S_z)}{2} \\ \large = \frac{\sum_{z \in (x,y)} (S_zn-S_z^2)}{2} \\ \large = \frac{n \cdot \sum_{z \in (x,y)} S_z - \sum_{z \in (x,y) } S_z^2 }{2} \] 考虑我们对 \(\large S\) 的定义中，只含一个路径上的点。由于这条限制，所以 \[ \large \forall z_1,z_2 \in (x,y) \quad S_{z_1} \cap S_{z_2} = \phi \] 所以得到总贡献 \[ \large \sum_{z \in (x,y)} S_z = n \Longrightarrow \frac{n^2 - \sum_{z \in (x,y) } S_z^2 }{2} \] 问题转化为最小化 \(\large \sum_{z \in (x,y) } S_z^2 = \Delta\)，可以 DP。

 

先考虑 \(\large S_z\) 如何求出。

设 \(\large sz_x\) 以 \(\large x\) 为根的子树大小，画图不难发现 \[ \large S_y = sz_x-sz_y \quad y \in son(x) \] 懒得证了，挺显然的。

设 \(\large f_x\) 为 添加路径是 \(\large x\) 到以 \(\large x\) 为根子树内任意一点，满足上式最小的贡献。 \[ \large f_x = \min_{y \in son(x)} { \{ f_y + (sz_x-sz_y) \} } \]  

这样就可以计算 \(\large \Delta\) 了。 \[ \large \Delta = \min_{y,z \in son(x)} { \{ f_y+f_z+(n-sz_y-sz_z)^2 \}} \] 显然可以斜率优化，但问题在于有两个需要维护的值。

不难想到，我们贪心地选择 \(\large y\)，维护最优的 \(\large z\)。

然后就是套路推式子。

但是个人感觉本题并不容易将转移变形成一次函数的形式，直接套两个决策 \(\large z_1,z_2\) 就行，最后推出 \[ \large \frac{(f_{z_1}+sz_{z_1}^2)-(f_{z_2}+sz_{z_2}^2)}{sz_{z_1}-sz_{z_2}} \le 2 \cdot (n-sz_y) \] 不难发现，如果 \(\large sz_y\) 单调增，那么对应的直线的斜率单调减，所以按这个排序。

对于两个决策 \(\large k_1,k_2\)，且前者优于后者，那么一定有 \[ \large \frac{f_{k_1}+sz_{k_1}^2}{sz_{k_1}} \le \frac{f_{k_2}+sz_{k_2}^2}{sz_{k_2}} \] 斜率是单调减的，决策集合就是一个上凸壳，单调队列维护即可。

 

最后讨论一下复杂度。

并不好表达，因为排序和斜率优化都是与子节点个数成正比的。

考虑每个点子节点个数的平均值 \(\large \xi\)。

这个字母能表示随机变量，直接拿来用了（

如果树是一条链，那么每个点的子节点和为 \[ \large \sum_{i=1}^{n-1} i = C_n^2 = \frac{n(n-1)}{2} \] 则 \[ \large \xi_1 = \frac{n-1}{2} \] 如果树是一朵菊花，那么只有根有 \(\large n-1\) 个子节点。 \[ \large \xi_2 = \frac{n-1}{n} \] 接着考虑普通树。

对于一颗普通树，不能产生贡献的叶子节点一定多于链，少于菊花。减少根的直接子节点的个数（其实就是叶子节点个数）一定会导致子节点个数和增大，反之只会减小。

所以，对于普通树 \[ \large \xi \in (\xi_2,\xi_1) \] 总复杂度 \(\large O(n \xi_n^2 \log_2 \xi_n)\)

实际上挺快的。

#include<cstdio>
#include<iostream>
#include<algorithm>
using namespace std;
#define ll long long
const int N=5e5+6;
ll n, dlt=(1ll<<60), sz[N], p[N], q[N], f[N];
ll c, h[N], ver[N<<1], nxt[N<<1];
inline void add(ll x,ll y) { ver[++c]=y, nxt[c]=h[x], h[x]=c; }
inline bool cmp(ll x,ll y) { return sz[x]<sz[y]; }
inline ll squ(ll x) { return x*x; }
inline ll Min(ll x,ll y) { return x<y? x:y; }
inline ll X(ll x) { return sz[x]; }
inline ll Y(ll x) { return f[x]+squ(sz[x]); }
inline double calc(ll x,ll y) {
    if(X(x)!=X(y)) return 1.0*(Y(x)-Y(y))/(X(x)-X(y));
    return Y(x)>Y(y)? 1e9:-1e9;
    // 横坐标相同，那么如果 x 纵坐标更大，则其斜率更优，保留。反之舍去
}
void dp(ll x,ll fr) {
    ll i, y, cnt=0, l=1, r=0;
    sz[x]=1;
    for(i=h[x],y;i;i=nxt[i]) if(ver[i]!=fr) {
        y=ver[i];
        dp(y,x), sz[x]+=sz[y];
    }
    f[x]=squ(sz[x]);
    if(sz[x]==1) return;
    for(i=h[x];i;i=nxt[i]) if(ver[i]!=fr) {
        y=ver[i], p[++cnt]=y;
        f[x]=Min(f[x],f[y]+squ(sz[x]-sz[y]));
    }
    dlt=Min(dlt,f[x]+squ(n-sz[x]));
    sort(p+1,p+cnt+1,cmp), q[++r]=p[1]; // 最小的一定是决策之一
    for(i=2;i<=cnt;++i) {
        y=p[i];
        while(l<r&&calc(q[l+1],q[l])<=2.0*(n-sz[y])) ++l;
        dlt=Min(dlt,f[y]+f[q[l]]+squ(n-sz[q[l]]-sz[y]));
        while(l<r&&calc(y,q[r-1])<=calc(q[r],q[r-1])) --r;
        q[++r]=y;
        // 一定都是小于等于，不然 WA
    }
}
int main() {
    scanf("%lld",&n);
    for(ll i=1,x,y;i<n;++i) scanf("%lld%lld",&x,&y), add(x,y), add(y,x);
    dp(1,0);
    printf("%lld\n",n*(n-1)/2+(n*n-dlt)/2);
}
