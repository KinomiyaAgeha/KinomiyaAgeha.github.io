luogu7914 括号序列 题解

发表于 2022-04-13 更新于 2022-06-28 分类于 题解 本文字数： 3.4k 阅读时长 ≈ 7 分钟

分析

明显是一个区间 DP，但是合法串情况比较多，所以要仔细分类。

设 f(l,r)f(l,r)f(l,r) 为区间 [l,r][l,r][l,r] 内的合法串的数量，q(l,r)q(l,r)q(l,r) 为 [l,r][l,r][l,r] 能否变成不超过 kkk 个*组成的串。

以下转移都是在满足 lll 能够成为左括号，rrr 能够成为右括号的前提下。

 

  1. ()

显然，特判 l+1=rl+1=rl+1=r 就好了。

1→f(l,r)1 \rightarrow f(l,r) 1→f(l,r)

  1. (S)

这种情况只会发生在 l+1≠rl+1 \neq rl+1​=r 的时候。左右括号都已经确定了，只要里面的能构成一整个长度合法的*串，就能贡献 1 个方案数。而 [l+1,r−1][l+1,r-1][l+1,r−1] 不管怎么样都能产生 f(l+1,r−1)f(l+1,r-1)f(l+1,r−1) 的贡献。

f(l+1,r−1)+q(l+1,r−1)→f(l,r)f(l+1,r-1) + q(l+1,r-1) \rightarrow f(l,r) f(l+1,r−1)+q(l+1,r−1)→f(l,r)

  1. (SA)

枚举S最后一个字符的位置 kkk，如果 q(l,k)=1q(l,k)=1q(l,k)=1，那么就转移。

q(l,k)⋅f(k+1,r)→f(l,r)q(l,k) \cdot f(k+1,r) \rightarrow f(l,r) q(l,k)⋅f(k+1,r)→f(l,r)

  1. (AS)

与上一步相同，枚举 AAA 的最后一个字符的位置 kkk，可以与上一步一起转移。

f(l,k)⋅q(l+1,k)→f(l,r)f(l,k) \cdot q(l+1,k) \rightarrow f(l,r) f(l,k)⋅q(l+1,k)→f(l,r)

  1. AB

枚举断点 kkk，暴力转移。

f(l,k)⋅f(k+1,r)→f(l,r)f(l,k) \cdot f(k+1,r) \rightarrow f(l,r) f(l,k)⋅f(k+1,r)→f(l,r)

且慢！

()()()

这个串是不是被重复计算了？[1,2][1,2][1,2] 与 [3,6][3,6][3,6]，[1,4][1,4][1,4] 与 [5,6][5,6][5,6]。

至理名言：当你想到一个 fAKe 的思路时，想想它为什么是 fAKe 的。

由于从不同的断点断开组成了相同的串，所以重复计算了。

如何避免？如果强制让 AAA 是由上面 4 种转移方式得到的串，那么 [1,4][1,4][1,4] 这个区间就不合法了。因为以上 4 种方法要求左右端点组成()，所以()()里面的)(是不合法的，因此 [1,4][1,4][1,4] 这个种断开方法不会产生贡献。如果不这样，那么()()可以看作A=()，B=()组成的AB形式的合法串，就重复了。

所以设 g(l,r)g(l,r)g(l,r) 为前 4 种转移对 f(l,r)f(l,r)f(l,r) 产生的贡献，能够在处理完后直接求出。

转移

g(l,k)⋅f(k+1,r)→f(l,r)g(l,k) \cdot f(k+1,r) \rightarrow f(l,r) g(l,k)⋅f(k+1,r)→f(l,r)

  1. ASB

同上，枚举 AAA 的最后一位 iii 和 BBB 的第一位 jjj。

那么 A=[l,i]A = [l,i]A=[l,i]，S=[i+1,j−1]S=[i+1,j-1]S=[i+1,j−1]，B[j,r]B[j,r]B[j,r]。

转移

∑i=l+1i<r∑j=i+2j<rg(l,i)⋅q(i+1,j−1)⋅f(j,r)\sum_{i=l+1}^{i<r} \sum_{j=i+2}^{j<r} g(l,i) \cdot q(i+1,j-1) \cdot f(j,r) i=l+1∑i<r​j=i+2∑j<r​g(l,i)⋅q(i+1,j−1)⋅f(j,r)

单是这个转移就是 O(n2)O(n^2)O(n2) 的，考虑优化。

首先，S这个串必须是连续的，而只有能够出现一个S才能产生贡献。所以，能够产生贡献的一定是一个连续的区间。（想象一下“滑动窗口”）。

所以一定能找到一个 ppp，满足 ∀k∈[i,p]\forall k \in[i,p]∀k∈[i,p]，都有 q(i,k)=1q(i,k)=1q(i,k)=1。

所以提一下公因式 g(l,i)g(l,i)g(l,i)，得到

∑i=l+1i<r(g(l,i)⋅∑j=i+2j≤pf(j,r))\sum_{i=l+1}^{i<r} \Big( g(l,i) \cdot \sum_{j=i+2}^{j \le p} f(j,r) \Big) i=l+1∑i<r​(g(l,i)⋅j=i+2∑j≤p​f(j,r))

关于 jjj 的那一项是能够 O(1)O(1)O(1) 计算的。右端点固定，左端点不同，可以使用后缀和优化。这个可能不太熟悉，但就是把前缀和对 [1,k][1,k][1,k] 求和改为对 [k,n][k,n][k,n] 求和。

设 SkS_kSk​ 为 [k,r][k,r][k,r] 这一段的和，于是转移就有了

∑i=l+1i<r(g(l,i)⋅(Si+2−Sp+1))→f(l,r)\sum_{i=l+1}^{i<r} \Big( g(l,i) \cdot (S_{i+2}-S_{p+1} ) \Big) \rightarrow f(l,r) i=l+1∑i<r​(g(l,i)⋅(Si+2​−Sp+1​))→f(l,r)

 

最终答案 f(1,n)f(1,n)f(1,n)

CODE

#include<bits/stdc++.h>
using namespace std;
#define ll long long
const int N=505, mod=1e9+7; 
int n, k;
char s[N];
ll f[N][N], g[N][N], q[N][N], sum[N];
bool check(int l,int r) {
	if(s[l]=='('&&(s[r]==')'||s[r]=='?')) return 1;
	if(s[l]=='?'&&(s[r]==')'||s[r]=='?')) return 1;
    // 判断是否合法
	return 0;
}
int main() {
	scanf("%d%d%s",&n,&k,s+1);
	for(int l=1;l<=n;++l) for(int r=l;r<=n;++r) {
		if(s[r]!='*'&&s[r]!='?') break;
		if(r-l+1>k) break; // 长度不超过k
		q[l][r]=1;
	}
	for(int len=2;len<=n;++len) for(int l=1;l+len-1<=n;++l) {
		int r=l+len-1;
		if(check(l,r)) {
			if(l+1==r) (f[l][r]+=1)%=mod; // 1
			else (f[l][r]+=f[l+1][r-1]+q[l+1][r-1])%=mod; // 2
			for(int k=l+1;k<=r-2;++k) {
				(f[l][r]+=f[l+1][k]*q[k+1][r-1]+f[k+1][r-1]*q[l+1][k])%=mod;
                // 3和4放到一块转移了
			}
			g[l][r]=f[l][r];
			for(int k=l+1;k<=r-2;++k) (f[l][r]+=g[l][k]*f[k+1][r]%mod)%=mod;
            // 5
			memset(sum,0,sizeof(sum));
			for(int k=r;k>=l;--k) (sum[k]+=sum[k+1]+f[k][r])%=mod;
            // 后缀和数组
			int p=0;
			for(int k=l+1;k<=r-2;k++) {
				p=max(p,k+1);
				while(q[k+1][p]) ++p; 
				(f[l][r]+=g[l][k]*(sum[k+2]-sum[p+1])%mod)%=mod;
                //就和前缀和数组求区间和要a[r]-a[l-1]一样，这里p要+1
			}
		}
	}
	printf("%lld\n",(f[1][n]+mod)%mod);
}
