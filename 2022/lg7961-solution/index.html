首先明确 ai∈[0,m]a_i \in [0,m]ai​∈[0,m]。

设 f(x,S)f(x,S)f(x,S) 为当前 {ai}\{a_i\}{ai​} 有 $ x$ 项，其中 S=∑i=1x2aiS = \sum_{i=1}^x 2^{a_i}S=∑i=1x​2ai​，还能够产生的权值和。

状态总数为 O(2mnm)O(2^{m}nm)O(2mnm)，只要保证转移在 O(m)O(m)O(m) 之内实现就可行。

这个状态明显是记忆化搜索。

如果多选 ax+1=ka_{x+1}=kax+1​=k，那么必然 k∈[0,m]k \in [0,m]k∈[0,m]，同时 S+2kS+2^kS+2k，状态变成了 f(x+1,S+2k)f(x+1,S+2^k)f(x+1,S+2k)，对 f(x,S)f(x,S)f(x,S) 产生的贡献为 f(x+1,S+2k)⋅vkf(x+1,S+2^k) \cdot v_kf(x+1,S+2k)⋅vk​。这个状态的每一个前继状态都必定要在算权值的时候乘上 vkv_kvk​，由于乘法结合律与分配律，只要边界设置对了，这样做就是对的。枚举 kkk，转移 O(m)O(m)O(m)。

f(x,S)=∑i=0mf(x+1,S+2m)⋅vif(x,S) = \sum_{i=0}^m f(x+1,S+2^{m}) \cdot v_i f(x,S)=i=0∑m​f(x+1,S+2m)⋅vi​

然后就是边界，如果上面我们这么干，那么必须 x=nx=nx=n 且 1 的个数小于 kkk 的状态的贡献为 1，否则为 0。

最终答案 f(0,0)f(0,0)f(0,0)

CODE

#include<bits/stdc++.h>
using namespace std;
#define ll long long
const int N=35, M=105;
const ll mod=998244353;
int n, m, k, v[N];
ll f[N][123000];
int read() {
	int a=0; char c=getchar();
	for(;!isdigit(c);c=getchar());
	for(;isdigit(c);a=a*10+(c^48),c=getchar());
	return a;
}
int count(ll x) {
	int cnt=0;
	while(x) cnt+=x&1, x>>=1;
	return cnt;	
}
// 统计x中1的个数
ll dp(int x,ll s) {
	if(x>=n) return count(s)<=k;
	if(f[x][s]!=-1) return f[x][s];
	ll dlt=0;
	for(int i=0;i<=m;++i) (dlt+=dp(x+1,s+(1<<i))*v[i])%=mod;
	return f[x][s]=dlt;
}
int main() {
	n=read(), m=read(), k=read();
	for(int i=0;i<=m;++i) v[i]=read();
	memset(f,-1,sizeof(f));
	printf("%lld\n",dp(0,0));
}

正解

分析

暴力状态无论如何优化都无法承受了，考虑重新设计状态。

暴力状态瓶颈的原因在与状态数量太多，十进制下 SSS 实在是太大了。那么可以试着按照二进制位设计状态。

建议好好品一下样例解释 #1……

当一个 aia_iai​ 确定时，只会发生 2 种情况。

  1. aia_iai​ 这个数首次出现，那么 SSS 的二进制中 1 的个数增加 1。
  2. aia_iai​ 这个数之前出现过了，那么 SSS 的二进制中那一位就要向前一位进 1，同时变成 0。由于不知道下一位是啥所以无法预测 1 的数量变化。

但是对于任何一位，他只能向更高的一位进，至于最后到了那里不重要，可以到更高位的时候再考虑。从高位状态转移到低位状态无后效性，还是记忆化搜索。

那么一个神级状态就要产生了！

f(k,i,x,y)f(k,i,x,y)f(k,i,x,y)，表示已经确定了 SSS 二进制的 kkk 位，其中 {a}\{a\}{a} 数列确定了 iii 项，有 xxx 个 1，同时第 k−1k-1k−1 位进了 yyy 个 1 到 kkk 位，还能够产生的贡献。

为什么这么设计呢？首先它能够唯一确定地表示 SSS 与 {a}\{a\}{a}，并且状态数量级别是 O(mn3)O(mn^3)O(mn3)。要知道测试点中 m∈[7,100]m \in [7,100]m∈[7,100]，n∈[5,30]n \in [5,30]n∈[5,30]，这是相当大的优化了。而且样例的做法启发我们用组合数求出方案，用 vkv_kvk​ 的不同次方来统计贡献。

kkk 其实隐式代表着 vkv_kvk​。转移的时候枚举 j∈[0,n−i]j \in [0,n-i]j∈[0,n−i]，表示剩下的 n−in-in−i 项中，kkk 用几项。那么从 n−in-in−i 个位置里任选 jjj 个为 kkk，方案数为 Cn−ijC_{n-i}^jCn−ij​，都会产生 vkjv_k^jvkj​ 的贡献。

最关键的是剩下两个状态如何表示。

上一位进了 yyy 个 1，这一位多了 jjj 个 1，二进制下满二进一，那么就是进 ⌊y+j2⌋\lfloor \frac{y+j}{2} \rfloor⌊2y+j​⌋ 个 1 到 k+1k+1k+1 位。

由于规定了进位顺序，当 y+jy+jy+j 是奇数时，kkk 位上必定留下 1，x+1x+1x+1，否则 xxx 不变。

好的！转移 O(n)O(n)O(n)！

f(k,i,x,y)=∑j=0n−if(k+1,i+j,x+d(y+j),⌊y+j2⌋)⋅Cn−ij⋅vkjf(k,i,x,y) = \sum_{j=0}^{n-i} f(k+1,i+j,x+d(y+j),\lfloor \frac{y+j}{2} \rfloor) \cdot C^j_{n-i} \cdot v_k^j f(k,i,x,y)=j=0∑n−i​f(k+1,i+j,x+d(y+j),⌊2y+j​⌋)⋅Cn−ij​⋅vkj​

当 ttt 为奇数时，d(t)d(t)d(t) 为 1，反之为 0。

边界依然很重要。当 k>mk>mk>m 时，不存在这样的 SSS，这个状态直接返回 0。

当 i=ni=ni=n 的时候，此时 [1,k][1,k][1,k] 位上有 xxx 个 1，前面进了 yyy 个 1。由于 (k,m](k,m](k,m] 此时全为 0，那么 yyy 个 1 最后产生的 1 数量是它的二进制中 1 的个数。所以就像暴力方法一样，如果 1 的数量合法，那么返回 1，否则返回 0。

答案 f(0,0,0,0)f(0,0,0,0)f(0,0,0,0)

复杂度 O(mn4)O(mn^4)O(mn4)

CODE

#include<bits/stdc++.h>
using namespace std;
#define ll long long
const int N=35, M=105;
const ll mod=998244353;
int n, m, kk;
ll c[N][N], v[M][N], f[M][N][N][N];
// c[i][j]是组合数，v[i][j]是v[i]的j次幂
ll read() {
	ll a=0; char c=getchar();
	for(;!isdigit(c);c=getchar());
	for(;isdigit(c);a=a*10+(c^48),c=getchar());
	return a;
}
int count(ll x) {
	int cnt=0;
	while(x) cnt+=x&1, x>>=1;
	return cnt;	
}
ll dp(int k,int i,int x,int y) {
	if(i>=n) return x+count(y)<=kk;
	if(k>m) return 0;
	if(f[k][i][x][y]!=-1) return f[k][i][x][y];
	ll dlt=0;
	for(int j=0;j<=n-i;++j) (dlt+=dp(k+1,i+j,x+(y+j)%2,(y+j)/2)*c[n-i][j]%mod*v[k][j]%mod)%=mod;
	return f[k][i][x][y]=dlt;
}
int main() {
	n=read(), m=read(), kk=read();
	for(int i=0;i<=m;++i) v[i][1]=read();
	c[0][0]=1;
	for(int i=1;i<=n;++i) {
		c[i][0]=c[i][i]=1;
		for(int j=1;j<i;++j) c[i][j]=(c[i-1][j]+c[i-1][j-1])%mod;
	}
	for(int i=0;i<=m;++i) {
		v[i][0]=1;
		for(int j=2;j<=n;++j) v[i][j]=(v[i][j-1]*v[i][1])%mod;
	}
    // 预处理组合数与次幂
	memset(f,-1,sizeof(f));
	printf("%lld\n",dp(0,0,0,00));
}
