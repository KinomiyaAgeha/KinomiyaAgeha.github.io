luogu3651 展翅翱翔之时 题解

link

首先明确，如果将从每颗卫星的接收源向它自己连接一条有向边的话，那么题目中给出的是一个外向基环树森林。

可以修改每个节点的入边，并求使其强连通的最小代价。

基环树题目的核心在于那个环。外向树中，每个点最多有 1 条入边，所以存在的那个环就成了一个相对封闭的区域。

在代价最小的解中，断环为链时，每棵基环树的环只会改一条边。

简单证明：如果环上改了超过一条边，那么不能保证环上的点连通。假设修改了超过一条边，若环不连通，要花费更多的代价来使环联通，显然是不划算的；若环连通，那么显然只断一条边时仍然连通，也不划算。

设修改每条边的代价为此边的权。

如果所有基环树的环都被断开了一条边，使其强连通代价最小的方案为保留最长链并尽可能将节点首尾相连，而其他的边一定要改。

简单证明：如果最长链上的边被修改了，那么由于它是一条链，修改后必然要花更多的代价使它相连，并且修改最长链上的边的代价要大于修改其他的边。而首尾相连则是显然的。

实际上是最长外向链，注意我们并不需要求出这条链，讨论它的作用是完善思路，以外的边权累加。

根据上面的推断，我们最终得到的是一个环。环是满足强连通的最简单的结构。

设 f(i)f(i)f(i) 为节点 iii 的入边。

scanf("%lld",&n);
for(int i=1;i<=n;++i) {
	scanf("%lld%lld",&f[i],&x);
	add(f[i],i,x);
}

接着就是找环，用 cy(k)cy(k)cy(k) 表示环上第 kkk 个节点。

v(i)=0v(i)=0v(i)=0 表示节点 iii 没有被访问，v(i)=1v(i)=1v(i)=1 表示节点 iii 被访问过，但不是环上的点，v(i)=2v(i)=2v(i)=2 表示节点 iii 是环上的点。

void find(int x) {
    // 对于任意起点i，这个函数都能找到环的。
    // 因为通过找父亲的方法一定能到达环上
    // 画图就能发现，从第一个v[x]=1的x再找父亲一遍一定能标记整个环
	k=0;
	while(!v[x]) v[x]=1, x=f[x];
	while(v[x]==1) cy[++k]=x, v[x]=2, x=f[x];
}

然后找每棵树的最长链。设 out(i)out(i)out(i) 为节点 iii 权值最大出边。

对环上的每个点进行 dfs，不断更新 out(i)out(i)out(i)，并将非最长链上的边的权值累加 ansansans。

void dfs(int x,int fa) {
	if(!v[x]) v[x]=1;
	for(int i=h[x];i;i=nxt[i]) {
		int y=ver[i];
		if(y==fa||v[y]==2) continue;
		// 不访问环上节点 
		dfs(y,x);
		if(w[out[x]]>w[i]) ans+=w[i]; else ans+=w[out[x]], out[x]=i;
		//因为要贪心保留最长链，而最长链是由若干节点的最大边权构成的，所以不必累加边权 
		//w[out[x]]<w[i]，说明此时out[x]还不是最大出边 
    }
}

接着就是环上点的处理，我们考虑两种情况。

对于环上的点 iii，要将环拆为链。

  1. 断开 f(i)f(i)f(i) 到 iii 的边。

  2. 断开 out(f(i))out(f(i))out(f(i))。

对于这两种情况，其代价分别用 xxx 与 yyy 维护。

xxx 表示未断环，转移操作 2 的代价，累加边权。

yyy 表示已将环断开一条边的代价，可以由两种操作转移而来。

又因为操作 1 又可以通过 xxx 或 yyy 两种状态转移，所以就有了这个奇怪的式子。

最后对于每棵树，累加 yyy 就可以了。

for(int i=1;i<=n;++i) if(!v[i]) {
	find(i);
	if(k==n) { puts("0"); return 0; }
    /* 如果环上的点等于n，那么就已经在同一个环里 */
	int x=0, y=1<<30;
	for(int j=1;j<=k;++j) dfs(cy[j],0);
	for(int j=1;j<=k;++j) {
		y=min(min(x,y)+w[cy[j]],y+w[out[f[cy[j]]]]);
		x+=w[out[f[cy[j]]]];
	}
	ans+=y;
}
